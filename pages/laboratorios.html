Proyecto desarrollo para laboratorios médicos

Giro: Laboratorios médicos

Objetivo: Implementar chatbot informativo para estudios clínicos, cotizacion e información de las sucursales

Tecnologías: Javascript, API REST, MySQL

La implementación del chatbot tenia como objetivo dirigirse a usuarios finales buscanan información referente a los horarios de las sucursales así como su ubicación y detalles sobre los estudios clínicos.

Se realiza la evaluación del proyecto con el equipo de PM y UX con el objetivo de entender los alcances y posteriormente hacer el diseño que vaya acorde con las necesidades del usuario, una vez aprobado por el cliente se procede a evaluar el desarrollo, solicitar los recursos indisepensables para evitar contratiempos en el desarrollo, por ejemplo web services API para conectarnos y posteriormente se inicia con el desarrollo en la plataforma Calixta de Auronix, basado en Javascript, se implementa el frontend mostrando en el menú principal las opciones referentes a los estudios clínicos, preparaciones, citas, horarios y costos.

Este proyecto en sí, no contaba con integración a webservices del cliente sin embargo si hubo un backend el cual se encargaba de extraer la información de una BD MySQL interna que se actualizaba de acuerdo a los datos del cliente. Además se construyó un sistema para que el usuario al compartirnos su ubicación, logremos encontrar ubicaciones de sucursales en un radio aproximado de 5km a la redonda, este fue un servicio interno creado y a su vez homologado para futuros proyectos.

La implementación contaba con un sistema de reporteria en el cual se desarrolla un funnel para el chatbot con el propósito de obtener data de los puntos más frecuentes por los que los usuarios navegan dentro del Bot, este se genera como un reporte de envío automático a mes vencido.

Código: De acuerdo a un NDA firmado por la empresa Auronix, no se permite revelar el contenido del código.
